Branch & Pull Request (Web-Only)

Any time we add pages or change text, we’ll do it on a branch so you can preview and revert easily.

  1. Create a branch while uploading
    • Open your repo on GitHub → Add file → Upload files → drag the new files.
    • Below the file list, choose Create a new branch and name it like feature/offer-suite-v2.
    • Click Propose changes (or Commit changes).
  2. Open a Pull Request
    • GitHub will show “Compare & pull request.” Click it.
    • Add a short title, e.g., “Offer Suite v2.”
    • Click Create pull request.
  3. Preview via GitHub Pages (optional)
    • Pages can only publish one branch at a time. To preview without switching, open the PR and browse the changed files; or temporarily switch Pages to your branch, then switch back after testing.
  4. Merge
    • When happy, click Merge pull request → Confirm.
    • If your Pages is set to main, it will redeploy with the new files.
  5. Rollback if needed
    • Go to Pull requests → find the PR → click Revert (GitHub creates a new PR that undoes changes). Merge it.
We’ll call out “Branch here” every time we deliver a new ZIP.
